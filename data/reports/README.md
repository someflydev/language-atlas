# Atlas Analytics Reports

This directory contains the output of the Atlas Analytics suite, generated by `scripts/generate_reports.py`. These reports provide specialized, high-signal data artifacts from the Language Atlas SQLite database.

## Artifacts

### safety_complexity_trends.json
*   **Purpose:** Tracks the evolution of programming language safety and complexity over time.
*   **Metrics:**
    *   `avg_complexity`: A numerical mapping (Low: 1, Medium: 2, High: 3) of language complexity bias per decade.
    *   `safety_distribution`: A count of languages by their safety model (manual, runtime, compile_time, hybrid) for each decade.
*   **Interpretation:** Use this to visualize if languages are becoming safer or more complex over the decades.

### creator_impact.json
*   **Purpose:** Ranks the "architects" of the Language Atlas.
*   **Metrics:**
    *   `total_impact`: Sum of the influence scores for all languages a creator (person or organization) is credited with.
    *   `language_count`: Number of languages associated with the creator.
*   **Interpretation:** Identifies the most influential individuals and entities in the history of programming.

### cluster_genealogy.json
*   **Purpose:** Measures the cross-pollination of ideas between different language clusters.
*   **Metrics:**
    *   `internal`: Count of influences where both the source and target languages belong to the same cluster.
    *   `external`: Count of influences originating from this cluster but targeting a different cluster.
    *   `ratio`: The "Internal vs. External Influence" ratio.
*   **Interpretation:** High internal ratios suggest a specialized, self-referential ecosystem (e.g., Systems), while high external ratios suggest broad, foundational impact (e.g., Mathematics).

### innovation_trends.json
*   **Purpose:** Captures the dominant technological themes of each era.
*   **Structure:** Groups key innovations extracted from language profiles by their generation (e.g., dawn, early, web1, renaissance).
*   **Interpretation:** Serves as a "Keyword Cloud" of the core advancements that defined each stage of programming evolution.

### db_health.json
*   **Purpose:** Identifies structural anomalies and missing content in the database.
*   **Checks:**
    *   `orphans`: Languages with no recorded incoming or outgoing influences.
    *   `terminal`: Languages that were influenced by others but did not (yet) influence any subsequent languages in the Atlas.
    *   `high_influence_missing_profiles`: High-impact languages (Influence Score >= 5) that lack a corresponding deep-dive `language_profile`.
*   **Interpretation:** A maintenance tool for identifying gaps in the lineage graph or the documentation.

## Generation
To regenerate these reports, run the following command from the project root:
```bash
uv run scripts/generate_reports.py --report all
```
