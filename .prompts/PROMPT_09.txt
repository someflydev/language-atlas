You are a strict data engineer and QA specialist. Your task is to write validation scripts for the "language-atlas" project.

CONTEXT:
The "language-atlas" relies on flat JSON data (`data/languages.json`, etc.) and canonical Markdown narratives. As this system grows, we must ensure that our data conforms to the required schemas and that references between files are not broken.

TASK:
1. Create a `scripts/` directory.
2. Write a Python script `scripts/validate_data.py`.

CONSTRAINTS & REQUIREMENTS:
- The script must load `data/languages.json` and verify that EVERY language has the required keys: `name`, `year`, `creators`, `paradigms`, `cluster`, `generation`, `primary_use_cases`, `key_innovations`, `influenced_by`, `influenced`, `philosophy`, `mental_model`, `complexity_bias`, `safety_model`.
- The script must validate the ENUM values (e.g., `complexity_bias` must be low, medium, or high; `safety_model` must be manual, runtime, compile_time, or hybrid).
- The script must cross-reference `influenced_by` and `influenced` to ensure that referenced language names actually exist in the dataset.
- The script should exit with code 0 if everything is valid, and code 1 with clear terminal output if errors are found.
- Rely strictly on standard Python libraries (`json`, `sys`, `pathlib`).

COMMIT BEHAVIOR:
- Make the script executable (`chmod +x scripts/validate_data.py`).
- Commit the changes to git.
- Commit message: "chore: add python validation script for canonical JSON data"
