# PROMPT_82: Dark Matter Alias Layer Cleanup for Legacy Language Variants

## Objective

PROMPT_80 and PROMPT_81 improved the canonical language graph and cleaned up
unknown references in `languages.json`, but the dark matter audit still relies
on a separate reviewed alias layer for semantic merges and bucket overrides.

This prompt is a **dark-matter curation pass only**. Its goal is to improve
`data/.dark_matter_aliases.json` and `data/.dark_matter_canonicals.json` so
common legacy spellings, symbolic display forms, and filename-era variants
resolve to the correct canonical Atlas entities during dark-matter collection.

This prompt must **not**:

- edit `data/languages.json`
- add or rename profile documents
- change application/runtime code
- add hardcoded semantic alias logic to `scripts/dark_matter_audit.py`

Only edit the hidden dark matter metadata files and regenerate the dark matter
report at the end.

## Mandatory First Steps

1. Read `AGENT.md` and `CLAUDE.md`.
2. Load `.context/data-pipeline.md` and `.context/commands.md`.
3. Read `scripts/README.md` sections for `dark_matter_audit.py` and
   `dark_matter_alias_candidates.py`.
4. Read `data/.dark_matter_aliases.json` and `data/.dark_matter_canonicals.json`.
5. Run `make dark-matter` and save a copy of the current report to
   `tmp/PROMPT_82_dark_matter_before.json`.
6. Create `tmp/PROMPT_82_checklist.md` with a checkbox per task below.
7. Check off each item immediately after completing it.

## Task 1: Preserve the Existing Scope of the Alias Layer

Before editing anything, keep these rules explicit:

- `data/.dark_matter_aliases.json` is for **reviewed semantic merges**
  from noisy human-facing terms to a canonical display term.
- `data/.dark_matter_canonicals.json` is for **declared canonical terms**
  and bucket/type overrides used by the dark matter resolver.
- Mechanical cleanup such as stripping years or punctuation variants that the
  Python resolver already handles must stay in Python; do not duplicate that
  logic with unnecessary aliases.
- Do not add every language in `data/languages.json` to
  `.dark_matter_canonicals.json`. Only add canonicals that the hidden alias
  layer needs in order to resolve reviewed terms safely.

If a candidate alias is ambiguous, skip it and record the ambiguity in the
checklist instead of forcing a mapping.

## Task 5: Keep the Prompt Narrow

Do not broaden this prompt into a general dark matter sweep.

Specifically do **not**:

- add aliases for vague prose fragments like `"The AI Infrastructure Reaction"`
- add aliases for speculative concepts that are not clearly equivalent
- add canonicals for every missing language or concept in the report
- edit `scripts/dark_matter_audit.py` unless a test proves the hidden JSON
  files alone cannot express the reviewed mapping

If you discover broader curation opportunities, note them in the checklist but
leave them for a later prompt.
