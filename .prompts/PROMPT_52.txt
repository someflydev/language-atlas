## Task 1 â€” Fix Makefile invocations

Read the current `Makefile` carefully before making any changes.

### Rule: scripts in src/app/core/

Any Makefile target that invokes a file under `src/app/core/` must use:

    PYTHONPATH=src uv run python -m app.core.<module_name>

where `<module_name>` is the filename without `.py`. Do NOT use
`python3 src/app/core/<file>.py` â€” that uses the system Python and
misses the venv.

Currently broken targets:
- `build`: `python3 src/app/core/build_sqlite.py`
  â†’ `PYTHONPATH=src uv run python -m app.core.build_sqlite`
- `audit`: `python3 src/app/core/auditor.py`
  â†’ `PYTHONPATH=src uv run python -m app.core.auditor`

### Rule: scripts in scripts/

Determine the correct invocation by whether the script is zero-dependency
(stdlib only) or needs the uv-managed venv. Read the import block of each
script to verify before deciding.

- `scripts/dark_matter_audit.py`: stdlib only (json, os, re, string,
  pathlib). Zero-dep. Keep as: `python3 scripts/dark_matter_audit.py`

- `scripts/generate_docs.py`: imports from app.core (DataLoader,
  SiteBuilder) but does its own `sys.path.insert` to add `src/`. It
  needs the venv for third-party deps (fastapi, jinja2, etc.) but does
  not need PYTHONPATH set externally.
  Keep as: `uv run python scripts/generate_docs.py`

- `scripts/prep_pages.py`: stdlib only (json, shutil, subprocess, sys,
  pathlib). Zero-dep.
  Change: `uv run python scripts/prep_pages.py` â†’ `python3 scripts/prep_pages.py`

Verify the `pages` target: it currently calls `uv run python
scripts/prep_pages.py`. Change only that call; `build` and `site` which
are dependencies of `pages` will be fixed by Task 1 above.

### Rule: pytest

`pytest` is listed in `pyproject.toml` dependencies but is NOT
installed globally on the system. All pytest invocations must go
through `uv run`:

- `test` target:           `pytest -v --cov=src/app/core`
  â†’ `uv run pytest -v --cov=src/app/core`

- `test-intensive` target: `pytest -m intensive`
  â†’ `uv run pytest -m intensive`

Note: do NOT add `--cov=...` to test-intensive â€” the current form is
intentional (intensive tests are slow; coverage is tracked by the
regular test run).

## Task 4 â€” .gitignore: un-ignore dark_matter_todo.json

`generated-reports/` is currently fully gitignored. We want to track
`generated-reports/dark_matter_todo.json` so the TODO list evolves
visibly with the data.

In `.gitignore`, immediately after the `generated-reports/` line, add:

    !generated-reports/dark_matter_todo.json

This negation exception tells git to track that one file despite the
directory-level ignore.

After editing .gitignore, run:

    git add generated-reports/dark_matter_todo.json

to start tracking it (it already exists on disk from prior runs of
`make dark-matter`). Confirm it appears in `git status` as a new
tracked file.
