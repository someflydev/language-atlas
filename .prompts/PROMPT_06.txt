You are an expert CLI tool developer. Your task is to build a Python CLI to explore the "language-atlas" project.

CONTEXT:
The "language-atlas" project contains a rich JSON data model and Markdown narratives. Before building a web UI, we need a simple, zero-dependency (or minimal dependency) Python CLI to query and explore the historical data.

TASK:
1. Create `cli.py` in the root directory.
2. Implement a Command Line Interface that allows a user to explore languages, paradigms, timelines, and influences.

CONSTRAINTS & REQUIREMENTS:
- Use standard library `argparse` or a lightweight library like `click` (if using `click`, ensure a `requirements.txt` is created).
- Commands MUST include:
  - `list` (list all languages, optionally filtered by generation or cluster)
  - `info <language>` (pretty-print the mental model, paradigms, and philosophy of a language)
  - `influences <language>` (show what it influenced and was influenced by)
  - `paradigms` (list all paradigms)
- The CLI must read directly from the `data/` directory JSON files (`data/languages.json`, etc.).
- Do NOT overengineer. No complex state machines. Just flat, explicit Python functions that read JSON and format terminal output.
- Gracefully handle missing data or missing files (since the user might be running this without all data perfectly seeded).

COMMIT BEHAVIOR:
- Test the CLI with `python cli.py --help` (conceptually).
- Commit the changes to git.
- Commit message: "feat: implement python CLI for exploring language atlas data"
