You are a Senior Data Scientist and Language Historian. Your task is to implement the "Atlas Analytics" suite—an expansion of our insight-generation capabilities that produces specialized, high-signal reports from the Language Atlas SQLite database.

CONTEXT:
We currently have a basic `historical_insights.json` (generated by `src/app/core/insights.py`). We need to broaden this into a modular reporting system that captures the "Evolution of Programming" through hard metrics.

TASK:
Implement the following reporting modules within a new script `scripts/generate_reports.py` (or expand `src/app/core/insights.py` if more idiomatic):

1.  **Safety & Complexity Evolution (`safety_complexity_trends.json`)**:
    *   Aggregate average `complexity_bias` (mapped to numerical values 1-3) and `safety_model` distribution per decade.
    *   Goal: Visualize if languages are becoming safer or more complex over time.

2.  **Impact Leaderboard (`creator_impact.json`)**:
    *   Rank creators (people and organizations) by the total `influence_score` of all languages they are credited with.
    *   Goal: Identify the most influential "architects" of the Atlas.

3.  **Cluster Genealogy (`cluster_genealogy.json`)**:
    *   For each `cluster` (e.g., AI, Systems, Web), calculate the "Internal vs. External Influence" ratio.
    *   How often do Systems languages influence other Systems languages vs. influencing other clusters?

4.  **Innovation Keyword Cloud (`innovation_trends.json`)**:
    *   Extract common themes from `key_innovations` across different `generation` tags.
    *   Goal: Identify the "buzzwords" or dominant technologies of each era (e.g., "Types" in `renaissance`, "Objects" in `web1`).

5.  **Database Health Audit (`db_health.json`)**:
    *   Identify "orphan" languages (no influences) and "terminal" languages (influenced by many, but influence none).
    *   Check for high-influence languages that lack a corresponding `language_profile`.

CONSTRAINTS:
- Use standard library Python only.
- Output reports to `data/reports/`.
- Ensure the script handles the project root correctly.
- Use `argparse` to allow running specific reports or "all".

DOCUMENTATION:
Update `scripts/README.md` (created in a previous task) or create a new `data/reports/README.md` to explain how to interpret these data artifacts.
