Objective: Consolidate and enhance the programming paradigms data into a single, high-quality source file.

Currently, information about programming paradigms is fragmented across three files:
- `data/paradigms.json`
- `data/docs/paradigms/paradigm_matrix.json`
- `data/docs/paradigms/paradigms_reference.json`

Your task is to merge the data from these three files into a single, comprehensive file at `data/paradigms.json`, elevating its quality and depth to match that of `data/languages.json` and `data/people.json`.

Please execute the following steps:

1. **Analyze Existing Data**: Read the contents of `data/paradigms.json`, `data/docs/paradigms/paradigm_matrix.json`, and `data/docs/paradigms/paradigms_reference.json`.
2. **Consolidate and Enhance**: Merge the information for each paradigm into a single, unified JSON object. You must synthesize the data to create a rich profile for each paradigm containing the following fields:
   - `name`: The formal name of the paradigm.
   - `description`: A clear, concise explanation of the paradigm's core philosophy and mechanics.
   - `year_introduced`: The approximate year or era the paradigm was introduced or popularized (research and infer if necessary).
   - `motivation`: The historical context, software crisis, or specific problems the paradigm was created to solve.
   - `languages`: A list of notable programming languages that exemplify or heavily utilize this paradigm.
   - `connected_paradigms`: A list of related, complementary, or contrasting paradigms.
   - `key_features`: Any important/useful information, mechanics, traits, or nuances extracted from the two `data/docs/paradigms` files.
3. **Alphabetize**: Ensure the final JSON array is strictly alphabetized by the `name` field.
4. **Update File**: Write the consolidated, enhanced, and sorted JSON array back to `data/paradigms.json`.
5. **Verify**: Check the structural integrity of the new `data/paradigms.json` to ensure it is valid JSON and contains all the enriched fields.
6. **Cleanup**: Once the new `data/paradigms.json` is verified and complete, delete the redundant files:
   - `data/docs/paradigms/paradigm_matrix.json`
   - `data/docs/paradigms/paradigms_reference.json`
   (If the `data/docs/paradigms` directory becomes empty, you may delete it as well).
