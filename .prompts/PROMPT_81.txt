# PROMPT_81: Unknown Reference Cleanup and Full-Stack entity_type Integration

## Objective

This is the integration and validation prompt for the entity_type work started
in PROMPT_79 and PROMPT_80. It has three phases:

1. **Unknown reference cleanup** — resolve the remaining `influenced` /
   `influenced_by` references that the audit still flags as unknown. Remove
   obscure or irrelevant entries from influence chains; add minimal
   `languages.json` records for historically significant ones.

2. **Full-stack entity_type surfacing** — update `DataLoader`, FastAPI routes,
   the CLI, and the TUI to filter and display by `entity_type`, so the
   classification work is actually usable and not just latent in the database.

3. **Comprehensive validation** — confirm that the SQLite build is producing
   a well-formed, fully-linked database and that all application layers
   retrieve and present data correctly.

## Mandatory First Steps

1. Read `AGENT.md` and `CLAUDE.md`.
2. Load `.context/architecture.md`, `.context/data-pipeline.md`,
   `.context/database.md`, `.context/routes.md`, and `.context/commands.md`.
3. Run `make audit` and save the output to `tmp/PROMPT_81_audit_before.txt`.
4. Run `make build` first to ensure the database is current.
5. Create `tmp/PROMPT_81_checklist.md` with checkboxes per phase.
6. Check off each item immediately after completing it.

## Phase 2: Surface entity_type Through the Full Stack

### 2a. DataLoader — add `entity_type` filter to `get_all_languages()`

In `src/app/core/data_loader.py`, update `get_all_languages()` to accept an
optional `entity_type` parameter:

```python
def get_all_languages(
    self,
    ...,
    entity_type: Optional[str] = 'language',
) -> List[Dict[str, Any]]:
```

**Default value is `'language'`** — existing callers that do not pass this
argument will automatically exclude foundations and artifacts, which is the
correct behaviour for language-browse views (the year-range scatter plots and
the comparison tool should not include Jacquard Loom or Numpy).

In the JSON path, add:
```python
if entity_type:
    langs = [l for l in langs if l.get('entity_type', 'language') == entity_type]
```

In the SQLite path, add the WHERE clause variant:
```python
if entity_type:
    query += " AND COALESCE(entity_type, 'language') = ?"
    params.append(entity_type)
```

Also update `get_all_clusters()` to only return clusters for
`entity_type='language'` by default (pass `entity_type='language'`
internally), so the cluster browse list is not polluted with `historical`
entries that are now foundations:

```python
def get_all_clusters(self, entity_type: str = 'language') -> List[str]:
```

Add equivalent filtering in both JSON and SQLite paths.

### 2b. FastAPI — expose entity_type filtering on the language list route

In `src/app/app.py`, find the `/languages` route (the main language list page
and the filtered partial). Add an optional `entity_type` query parameter:

```python

async def get_languages(
    request: Request,
    ...,
    entity_type: Optional[str] = Query(None),
) -> Response:
```

When `entity_type` is provided, pass it through to `data_loader.get_all_languages()`.
When it is `None`, pass `entity_type='language'` to preserve the default behaviour.

Add three navigation links to the language list template (or to the Atlas
index page, whichever is most natural) so users can browse foundations and
artifacts:

- `/languages?entity_type=language` — programming languages (default)
- `/languages?entity_type=foundation` — historical and theoretical foundations
- `/languages?entity_type=artifact` — tools, libraries, runtimes, and formats

The template should display the `entity_type` tab or label so users know
which section they are browsing.

Also add a new `/api/entity-types` JSON route that returns a summary:

Update `API_GUIDE.md` to document the new query parameter and the new route.

### 3d. Database integrity spot-checks

Run the following queries and confirm they return expected results:

```python
import sqlite3, json
conn = sqlite3.connect('language_atlas.sqlite')
conn.row_factory = sqlite3.Row

# Entity type distribution
print("=== Entity type distribution ===")
for row in conn.execute(
    "SELECT COALESCE(entity_type,'language') as et, COUNT(*) as n "
    "FROM languages GROUP BY et ORDER BY et"
).fetchall():
    print(f"  {row['et']}: {row['n']}")

# Influence edge count — should be substantially higher than before PROMPT_79
print("\n=== Influence edges ===")
row = conn.execute("SELECT COUNT(*) as n FROM influences").fetchone()
print(f"  Total influence edges: {row['n']}")

# Cross-type influence examples (foundations -> languages)
print("\n=== Foundation -> language influence samples ===")
for row in conn.execute("""
    SELECT src.name as src, tgt.name as tgt
    FROM influences i
    JOIN languages src ON i.source_id = src.id
    JOIN languages tgt ON i.target_id = tgt.id
    WHERE src.entity_type = 'foundation' AND tgt.entity_type = 'language'
    LIMIT 10
""").fetchall():
    print(f"  {row['src']} -> {row['tgt']}")

# Artifact -> language influence examples
print("\n=== Artifact -> language influence samples ===")
for row in conn.execute("""
    SELECT src.name as src, tgt.name as tgt
    FROM influences i
    JOIN languages src ON i.source_id = src.id
    JOIN languages tgt ON i.target_id = tgt.id
    WHERE src.entity_type = 'artifact' AND tgt.entity_type = 'language'
    LIMIT 10
""").fetchall():
    print(f"  {row['src']} -> {row['tgt']}")

# Profile coverage for entity types
print("\n=== Profile coverage by entity_type ===")
for row in conn.execute("""
    SELECT COALESCE(l.entity_type,'language') as et,
           COUNT(l.id) as total,
           COUNT(lp.id) as with_profile
    FROM languages l
    LEFT JOIN language_profiles lp ON l.id = lp.language_id
    GROUP BY et ORDER BY et
""").fetchall():
    print(f"  {row['et']}: {row['with_profile']}/{row['total']} have profiles")

conn.close()
```
