## Content Guidance per Event

### `garmisch_conference.json`
- Date: 1968 (October 7-11, Garmisch, West Germany)
- Organized by NATO Science Committee; Peter Naur and Brian Randell
  edited the proceedings
- Named the discipline "software engineering" as a deliberate analogy
  to civil and mechanical engineering — implying rigor was possible
- Key figures: Peter Naur, Friedrich Bauer, Brian Randell, Edsger Dijkstra
- Directly produced: structured programming movement, formal methods push

### `compiler_revolution.json`
- Date: 1952-1964
- Arc: Autocode (Glennie, 1952) → A-0 (Hopper, 1952) → FORTRAN (Backus, 1957)
  → COBOL (1959) → ALGOL 60 (1960) → PL/I (1964)
- The core debate: can a compiler produce code as efficient as a human
  assembly programmer? FORTRAN proved it could (Backus's optimizer)
- Impact: programmers could now think in algorithms, not registers

### `structured_revolution.json`
- Date: 1968-1980
- Dijkstra's "Go To Statement Considered Harmful" (1968 CACM letter)
- Wirth's stepwise refinement; Hoare's axiomatic semantics
- Pascal (1970) as the practical vehicle for structured programming
- Result: loops, functions, blocks replaced unconditional jumps as
  the primitive control structure

### `macro_debugging_crisis.json`
- Date: Late 1960s-1970s
- Macro systems in LISP, assembly, and early PL/I created programs
  where the source text bore little resemblance to what executed
- Motivated: hygienic macros (later Scheme), structured modules,
  encapsulation as a response to macro chaos
- Connects to the Software Crisis

### `memory_safety_crisis.json`
- Date: 1988-present (peak: 2000-2015)
- Morris Worm (1988): first internet worm exploiting a buffer overflow
- Timeline: buffer overflow → stack smashing → heap corruption → use-after-free
- Response: GC languages (Java, C#), Valgrind, AddressSanitizer,
  and ultimately Rust's ownership model
- 70% of CVEs at Microsoft and Google traced to memory safety bugs

### `multicore_crisis.json`
- Date: 2005-2015
- Intel's 4GHz wall hit in 2004; clock speeds plateaued
- "The Free Lunch Is Over" (Herb Sutter, 2005)
- Response: Erlang resurgence, Go's goroutines, Rust's fearless
  concurrency, async/await patterns, ReactiveX, actor model revival
- Key tension: shared mutable state becomes the enemy; functional
  programming concepts mainstreamed

### `pc_revolution.json`
- Date: 1975-1985
- Altair 8800 (1975), Apple II (1977), VisiCalc (1979),
  IBM PC (1981), Macintosh (1984)
- BASIC shipped in ROM on every 8-bit computer; a generation learned
  to program without a CS degree
- Impact on languages: demand for interpreted, interactive, forgiving
  languages; logo for children; HyperCard for non-programmers

### `security_crisis.json`
- Date: 1998-2015
- SQL injection, cross-site scripting, buffer overflows as mass-scale
  attack vectors; Code Red worm (2001), Heartbleed (2014)
- OWASP Top 10 (2003) codified common vulnerabilities
- Response: injection-safe query frameworks (prepared statements),
  sandboxing, memory-safe languages, security-conscious language
  design (Rust, Go's stdlib)

### `war_effort.json`
- Date: 1939-1945
- Colossus (1943, UK, Bletchley Park): first programmable electronic
  computer, breaking Lorenz cipher
- Turing's Bombe: electromechanical, not electronic, but theoretically
  foundational; Turing's work on Enigma
- ENIAC (1945, US): electronic, general-purpose (though programmed by
  plugboard/switches, not stored program)
- Konrad Zuse's Z3 (1941, Germany): first Turing-complete programmable
  computer (electromechanical)
- Impact: accelerated computing by decades; brought state funding and
  urgency to a previously academic field
