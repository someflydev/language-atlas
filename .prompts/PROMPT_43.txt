Objective: Consolidate and elevate the historical era, timeline, and narrative data into a single, high-quality core data file (`data/eras.json`).

Currently, the historical narrative of programming languages is fragmented and contains varying levels of structure across the following locations:
- `data/eras.json` (Very basic metadata)
- `data/docs/era_summaries/` (Directory of JSON files with detailed summaries)
- `data/docs/timeline.json` (A timeline of events and eras)
- `data/docs/crossroads.json` (Pivotal historical moments and shifts)
- `data/docs/modern_reactions.json` (Architectural responses and modern paradigm shifts)

Your task is to blend, re-write, and join all of this rich content into a comprehensive, highly-structured `data/eras.json` file. This new file should serve as the definitive source of truth for the historical context of the language atlas.

Please execute the following steps:

1. **Analyze Existing Data**: 
   - Read the current `data/eras.json`.
   - Read all JSON files in `data/docs/era_summaries/`.
   - Read `data/docs/timeline.json`, `data/docs/crossroads.json`, and `data/docs/modern_reactions.json`.

2. **Synthesize and Map**:
   - Use the basic eras defined in `data/eras.json` (and the `era_summaries`) as your foundational list of eras.
   - For each era, synthesize a rich profile. You will need to logically map the events from `timeline.json`, the pivotal moments from `crossroads.json`, and the themes from `modern_reactions.json` to the appropriate era based on context, years, and themes. *Note: Modern reactions might largely map to the most recent eras, but their catalysts should be tied to the historical eras they are reacting to.*

3. **Restructure `data/eras.json`**:
   The new `data/eras.json` should be a JSON array of era objects. Each object MUST contain the following fields:
   - `name`: The formal name of the era (e.g., "The Object-Oriented Explosion").
   - `period`: The timeframe (e.g., "1975-1990").
   - `description`: A synthesized, high-quality summary combining the best parts of the existing descriptions and era summaries.
   - `catalyst`: The core conflict, hardware limitation, or "software crisis" that triggered this era (extracted from crossroads/summaries).
   - `key_innovations`: An array of major conceptual or architectural breakthroughs that occurred.
   - `timeline_events`: An array of significant events or language releases specific to this era (mapped from `timeline.json` and `crossroads.json`). Each event should have a `year` (if available) and `description`.
   - `reactions_and_legacy`: How this era influenced the future or what modern reactions it triggered (mapped from `modern_reactions.json` and the era summaries).

4. **Verify and Write**:
   - Ensure the synthesized data is valid JSON and strictly follows the schema outlined above. The text should be engaging, polished, and maintain the high quality expected of the core atlas data.
   - Write the consolidated array to `data/eras.json`.
   - Ensure the eras are sorted chronologically.

5. **Cleanup Redundant Files**:
   Once you have verified that `data/eras.json` contains all the synthesized high-quality data, delete the following files and directories (using `rm` and `rmdir`):
   - All files in `data/docs/era_summaries/`, then the directory itself.
   - `data/docs/timeline.json`
   - `data/docs/crossroads.json`
   - `data/docs/modern_reactions.json`

6. **Update Scripts (if necessary)**:
   Check if any Python scripts in `scripts/` (e.g., `dark_matter_audit.py`, `data_stats.py`, or `generate_docs.py`) are hardcoded to look for the deleted files or directories. Update them to use the new `data/eras.json` structure or remove the references safely to ensure the toolkit remains functional.
