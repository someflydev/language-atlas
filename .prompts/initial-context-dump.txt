You are designing a project called `language-atlas`.

This project is a deeply structured, explorable system that captures:

- the historical evolution of programming languages
- paradigm shifts and tradeoffs
- major inflection points in computing
- relationships between languages (influence, reaction, ecosystem)
- conceptual constructs (immutability, concurrency, type systems, etc.)
- teaching-oriented narratives for future learners in the AI era

This is NOT just documentation.

This is a hybrid system combining:
- structured JSON data
- markdown documentation
- HTMX + server-rendered web UI
- Plotly visualizations
- CLI exploration tools
- learning paths

---

CRITICAL DESIGN PRINCIPLES:

- Every artifact must be understandable in isolation
- Every megaprompt must be runnable in a fresh session
- No reliance on prior prompts or system prompts
- Flat, explicit structure over magic
- Minimal JavaScript (HTMX only)
- Python-first backend
- Markdown as a canonical narrative layer
- JSON as canonical data layer
- CLI + Web UI as complementary interfaces
- AI-friendly repo structure (clear, inspectable, deterministic)
