You are a Data Integrity Architect. Your task is to perform a "Dark Matter Audit" of the Language Atlas to identify every language, concept, and organization that is referenced but lacks a deep-dive profile.

CONTEXT:
Our system is powered by `people.json`, `languages.json`, and multiple profile directories: `data/docs/language_profiles/`, `data/docs/concept_profiles/`, and `data/docs/org_profiles/`. A "complete" Atlas means every contribution listed for a person, every key innovation listed for a language, and every founding entity has its own detailed profile.

TASK:
1.  **Crawl people.json**: Extract all names and contributions. People already in `people.json` must be excluded from the concept and organization gap lists.
2.  **Crawl languages.json**: Extract language names, influences, innovations, and creators.
3.  **Crawl Existing Profiles**: 
    - Scan ALL fields in `language_profiles/`, `concept_profiles/`, and `org_profiles/`, including narrative fields like `overview`, `historical_context`, and `legacy`.
    - **Narrative Mining**: Extract `**Bolded Terms**` as potential concepts and patterns like `at [Organization Name]` (where Organization Name is 1-4 capitalized words) as potential organizations.
4.  **Canonicalization & Deduplication**:
    - **Normalize**: Ignore case, unify dashes/spaces/underscores, and strip parentheticals.
    - **Prefix Stripping**: Strip leading "The " from concept names (e.g., "The Actor Model" -> "Actor Model").
    - **Org Suffixes**: Strip organizational suffixes like "Inc.", "Corp.", "Labs", and "Software" for deduplication (e.g., "Apple Inc." and "Apple" should collide).
5.  **Identify and Categorize Gaps**:
    - **Version Filtering**: Do not flag versioned languages if the base language already exists (e.g., "Ada 95" is satisfied by an "Ada" profile).
    - **Organization Detection**: Classify an entity as an organization if it matches a high-value list (Bell Labs, Google, Xerox PARC, etc.) or contains specific suffixes (University, Labs, Foundation, etc.). Use a word-count guard (max 5 words) to avoid misclassifying sentences as organizations.
    - **Conflict Resolution**: Prioritize categorization in the order of `languages` > `organizations` > `concepts`.
6.  **Build the TODO List**: Create a unified, deduplicated JSON file at `data/reports/dark_matter_todo.json`.

OUTPUT STRUCTURE:
The `dark_matter_todo.json` should be organized like this:
{
  "missing_language_profiles": ["ABC", "D", ...],
  "missing_concept_profiles": ["ACID Properties", "Homoiconicity", ...],
  "missing_org_profiles": ["Bell Labs", "Xerox PARC", ...],
  "ambiguous_references": []
}

CONSTRAINTS:
- Use standard library Python only.
- Implement a length/complexity guard: skip any potential entity that exceeds 60 characters or 8 words to avoid narrative "bleed-in."
- Categorize people missing from `people.json` separately or exclude them if they are clearly personal names rather than concepts.

DOCUMENTATION:
- Update `AGENT.md` to reflect that `data/reports/dark_matter_todo.json` is the authoritative source for the next implementation phase.
- Update `scripts/README.md` to document the `dark_matter_audit.py` tool.
