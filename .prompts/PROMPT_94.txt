# PROMPT_94: Advanced Analytics — Leverage Scores, Concept Diffusion, and Anomaly Detection

## Objective

The Graph Intelligence Layer now has a solid foundation: materialized closure
tables, a derived-data pipeline, API endpoints, and UI enrichment. This prompt
adds the advanced analytics tier (Phase 5).

Three features:

1. **Pedagogical leverage score** — a per-language score that captures how
   much "learning value per step" a language provides. A language with high
   leverage teaches you a lot about the ecosystem you're entering.

2. **Concept diffusion analysis** — which concepts spread most widely through
   the influence graph, and which languages were the primary carriers.

3. **Anomaly detection** — identifying structural anomalies: unusually deep
   chains, disconnected islands, and languages with suspiciously high or low
   reachability relative to their era.

These analytics are surfaced in the existing `/insights` page and through two
new CLI commands.

## Product Requirements

### 1. Pedagogical leverage score

A language's **leverage score** measures how well it serves as an entry point
to the influence graph: how many descendants it reaches per unit of learning
depth. A shallow language that reaches many others is high-leverage.

**Formula**:
```
leverage = reachability_score / max(max_ancestor_depth, 1)
```

Where:
- `reachability_score` = descendant_count from `language_descendants`
- `max_ancestor_depth` = deepest ancestor depth from `language_ancestry`

High leverage: many descendants, shallow ancestry (an accessible gateway).
Low leverage: few descendants or very deep ancestry (a specialist endpoint).

**Implementation**: add a method to `InsightGenerator`:

```python
def calculate_leverage_scores(self, limit: int = 30) -> List[Dict[str, Any]]:
    ...
```

This method must:

1. Query `language_descendants` to get `descendant_count` and
   `max_descendant_depth` per language.
2. Query `language_ancestry` to get `max_ancestor_depth` per language.
3. Compute `leverage = descendant_count / max(max_ancestor_depth, 1)` in Python.
4. Return the top `limit` languages by `leverage` descending.
5. Include fields: `name`, `display_name`, `entity_type`, `year`,
   `leverage_score`, `descendant_count`, `max_ancestor_depth`.
6. Round `leverage_score` to 2 decimal places.

Also add `GET /api/insights/leverage` route in `app.py`:
- `InsightGenerator` takes a raw `sqlite3.Connection`, not a DataLoader.
  Instantiate it as: `InsightGenerator(data_loader._get_connection())`.
- `InsightGenerator` is not currently imported in `app.py`. Add:
  `from app.core.insights import InsightGenerator`
- Call `InsightGenerator(data_loader._get_connection()).calculate_leverage_scores(limit=30)`.
- Return JSON list. Wrap in try/except `sqlite3.OperationalError` to handle
  the case where closure tables don't exist yet (return `[]`).

### 2. Concept diffusion analysis

Each language is associated with one or more paradigms. The influence graph
creates a "diffusion" channel: when language A influences language B, B
inherits awareness of A's paradigms.

**Concept diffusion score** for a paradigm P = number of distinct languages
in the corpus (including descendants via the closure table) that are
associated with P OR that were influenced by a language associated with P.

**Implementation**: add a method to `InsightGenerator`:

```python
def calculate_concept_diffusion(self) -> List[Dict[str, Any]]:
    ...
```

This method must:

1. For each paradigm P, find all languages directly tagged with P via
   `language_paradigms`.
2. For each such language L, count how many languages have L as an ancestor
   in `language_ancestry` (i.e., L "transmitted" the paradigm downstream).
3. `diffusion_score` = (count of languages directly tagged with P) +
   (count of unique languages that have at least one P-tagged language as
   an ancestor)
4. Also identify the **primary carrier**: the language tagged with P that has
   the highest `reachability_score` (most transitive descendants).
5. Return all paradigms sorted by `diffusion_score` descending.
6. Fields: `paradigm_name`, `direct_languages`, `diffusion_score`,
   `primary_carrier_name`, `primary_carrier_year`.

Add `GET /api/insights/concept-diffusion` route in `app.py` using the same
`InsightGenerator(data_loader._get_connection())` instantiation pattern and the
same `sqlite3.OperationalError` guard described for the leverage endpoint.

### 3. Anomaly detection

Add `InsightGenerator.detect_graph_anomalies()`:

```python
def detect_graph_anomalies(self) -> Dict[str, Any]:
    ...
```

Detect and return three anomaly categories:

**3a. Unusually deep chains**:
Find the top 5 longest influence paths in `language_lineage_paths_bounded`.
A "deep chain" is a path with depth >= 7.
Return: list of `{"path_text": "...", "depth": N}` sorted by depth descending.

**3b. High-connectivity outliers**:
Languages whose `reachability_score` is more than 2 standard deviations above
the mean for their `generation` (era). These are the "unexpectedly central"
nodes.
Return: list of `{"name": "...", "generation": "...", "reachability_score": N,
"generation_mean": N, "z_score": N}` sorted by z_score descending.

**3c. Era gaps**:
Find pairs of consecutive generations (e.g., "dawn" → "early", "early" →
"web1") where fewer than 3 languages in the later generation have ANY ancestor
in the earlier generation. These are generational disconnects in the graph.
Return: list of `{"from_generation": "...", "to_generation": "...",
"connected_count": N}` for pairs where `connected_count < 3`.

Add `GET /api/insights/anomalies` route in `app.py` using the same
`InsightGenerator(data_loader._get_connection())` pattern and error guard.

### Concept diffusion SQL optimization

Step 2 of `calculate_concept_diffusion` involves finding, for each paradigm,
how many languages have at least one P-tagged language as an ancestor. This
can be computed with a single SQL query:

```sql
SELECT p.name AS paradigm_name,
       COUNT(DISTINCT lp.language_id) AS direct_languages,
       COUNT(DISTINCT la.root_language_id) AS carrier_descendants
FROM paradigms p
JOIN language_paradigms lp ON lp.paradigm_id = p.id
LEFT JOIN language_ancestry la ON la.ancestor_language_id = lp.language_id
GROUP BY p.id, p.name
```

Compute `diffusion_score = direct_languages + carrier_descendants` in Python.
For `primary_carrier`: a second query or subquery per paradigm is acceptable
(there are only ~20 paradigms).

### Z-score computation

For the outlier detection, compute mean and standard deviation in Python
(using `statistics.mean` and `statistics.stdev` from the standard library).
Group languages by generation first, then compute per-generation statistics.
A language with fewer than 3 peers in its generation is excluded from
z-score analysis (insufficient sample).
