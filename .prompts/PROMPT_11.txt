You are an expert backend and web developer. Your task is to build the Web UI Foundation for the "language-atlas" project using a Python-first, HTML-first approach.

CONTEXT:
The "language-atlas" requires a web interface. We are strictly avoiding SPA frameworks (no React, Vue, etc.). The stack must be Python (FastAPI or Starlette), Jinja2 for server-rendered HTML, HTMX for interactions, and Plotly for visualizations.

TASK:
1. Create `app.py` (FastAPI or Starlette application).
2. Create or update `requirements.txt` containing at least `fastapi`, `uvicorn`, `jinja2`, `python-multipart`.
3. Create a `templates/` directory with a `base.html` and an `index.html`.
4. Create a `static/` directory with `static/css/styles.css` and `static/js/` (include the HTMX library via CDN in the template or download it).

CONSTRAINTS & REQUIREMENTS:
- The backend MUST use the `core/data_loader.py` utility which loads data from `data/languages.json`.
- `index.html` should display a list or grid of all languages loaded from the data layer.
- Static assets MUST be in `static/css/styles.css` and `static/js/`.
- Jinja2 templates MUST be in `templates/`.

COMMIT BEHAVIOR:
- Ensure the code is self-contained and handles potential missing data gracefully.
- Commit the changes to git.
- Commit message: "feat: establish FastAPI and HTMX foundational web UI"
