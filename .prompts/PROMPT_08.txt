You are an expert CLI tool developer. Your task is to build a Python CLI to explore the "language-atlas" project.

CONTEXT:
The "language-atlas" project contains a rich JSON data model and Markdown narratives. Before building a web UI, we need a simple, zero-dependency (or minimal dependency) Python CLI to query and explore the historical data.

TASK:
1. Create a `core/` directory.
2. Create `core/data_loader.py`: A Python utility to load and parse the `data/*.json` files into consistent objects.
3. Create `cli.py` in the root directory, importing and using `core/data_loader.py`.
4. Implement a Command Line Interface that allows a user to explore languages, paradigms, timelines, and influences.

CONSTRAINTS & REQUIREMENTS:
- `core/data_loader.py` should provide clean functions like `get_all_languages()`, `get_language(name)`, etc.
- Use standard library `argparse` or a lightweight library like `click`. If using `click`, ensure a `requirements.txt` is created or updated.
- Commands MUST include:
  - `list` (list all languages, optionally filtered by generation or cluster)
  - `info <language>` (pretty-print the mental model, paradigms, and philosophy of a language)
  - `influences <language>` (show what it influenced and was influenced by)
  - `paradigms` (list all paradigms)
  - `research <language>` (generate a Research Brief with high-intent discovery prompts for an LLM)
- Both `cli.py` and the future web app MUST use `core/data_loader.py` for data access.
- Gracefully handle missing data or missing files.

COMMIT BEHAVIOR:
- Test the CLI with `python cli.py --help` (conceptually).
- Commit the changes to git.
- Commit message: "feat: implement shared data loader and python CLI explorer"
